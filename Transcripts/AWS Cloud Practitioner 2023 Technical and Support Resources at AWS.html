AWS Cloud Practitioner 2023: Technical & Support Resources at AWS

Whenever Amazon Web Services gets a new customer to create even a basic account, they want to keep that customer for the lifetime of that entity. AWS supports the widest array of business and organizational customers in the world. Begin this course by exploring AWS support plans, including AWS Developer, AWS Business, AWS Enterprise On-Ramp, and AWS Enterprise. Then you will dive into AWS Professional Services and Solutions Architects, the AWS Partner Network, and the AWS Support Center. Next, you will learn how to locate helpful resources at AWS, like whitepapers, blogs, and documentation. Finally, you will investigate the roles of AWS Trusted Advisor, the AWS Health Dashboard, and the AWS Health API, focus on reporting abuse of AWS resources with AWS Trust and Safety, and find out how to use AWS Partners and the Marketplace. This course is one of a collection that prepares learners for the AWS Certified Cloud Practitioner (CLF-C02) exam.

After completing this video, you will be able to describe the role of the AWS Trust and Safety team to report abuse of AWS resources.

  • describe the role of the AWS Trust and Safety team to report abuse of AWS resources
[Video description begins] Topic title: AWS Trust and Safety. Presented by: Michael Shannon. [Video description ends]

Let’s take a moment to look at AWS Trust & Safety. The AWS Trust & Safety or T&S team is a global group that helps protect against abusive uses of AWS services.

It also functions to build trust with AWS customers, partners, and other stakeholders. The T&S team scrutinizes potential abuse cases and contacts AWS customers to stop damaging activities.

The AWS T&S team serves as the first line of defense by investigating trends and reporting findings to AWS service groups as necessary.

AWS Support Engineers, Solutions Architects, Technical Account Managers (TAMs), and product teams are available to provide guidance and help as needed. The AWS T&S team helps customers when their cloud resources are exploited to conduct abusive actions like spam, port scanning, denial-of-service (DoS) attacks, or various types of malware.
