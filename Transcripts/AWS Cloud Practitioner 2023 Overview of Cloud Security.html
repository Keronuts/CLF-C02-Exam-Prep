AWS Cloud Practitioner 2023: Overview of Cloud Security

Security and compliance are critical aspects of the cloud security model. These shared responsibilities must involve comprehensive due diligence and due care on the part of the cloud computing customer.

In this course, you will explore the benefits of cloud security, including secure scaling and automation. Then you will learn how to capture and locate logs associated with cloud security and find out how customers secure resources on AWS using Amazon Inspector. Next, you will find out how to work with AWS Security Hub and AWS Shield and outline the key features of AWS Guard Duty. Finally, you will compare AWS encryption options and look at AWS services that aid in governance and compliance, including AWS CloudWatch, AWS CloudTrail, AWS Audit Manager, and AWS Config.

This course is one of a collection that prepares learners for the AWS Certified Cloud Practitioner (CLF-C02) exam.

In the previous demonstration, we looked at the importance of using logs at Amazon Web Services. Another important vulnerability assessment tool at AWS is Amazon Inspector. Inspector is a managed vulnerability assessment service that continually scans AWS workloads for software vulnerabilities, gaps, and unintentional network exposures. It can automatically locate and scan running Elastic Compute Cloud (EC2) instances, container images in the Amazon EC2, the Container Registry, as well as AWS Lambda functions or serverless code for known vulnerabilities and unintended exposure. Amazon Inspector creates a finding when it discovers a software vulnerability or a network configuration issue.

A finding describes the vulnerability, it identifies the affected resource, it rates the severity of the vulnerability, and it provides remediation guidance not unlike the CVSS in the National Vulnerability Database. Customers can analyze results using the Amazon Inspector console or visualize and process discoveries through other associated AWS services. Amazon Inspector will centrally manage multiple Amazon Inspector accounts, continuously scan your environment for vulnerabilities and network exposure. Assess vulnerabilities accurately with a risk score in Common Vulnerabilities Scoring System (CVSS) format. Identify high-impact findings with the Amazon Inspector dashboard, and manage your findings using customizable views. After completing this video, you will be able to outline key features of AWS GuardDuty.

[Video description begins] Topic title: AWS GuardDuty. Presented by: Michael J. Shannon. [Video description ends]

AWS GuardDuty. This is a managed threat detection service that constantly monitors for malevolent and unauthorized behavior. It watches out for unusual application programming interface calls (APIs), or potentially unauthorized deployments that indicate a possible account compromise or a zero day. Zero being the first day, it shows up in the wild. GuardDuty can detect potentially compromised instances or reconnaissance attacks by advanced persistent threat actors. The one thing that GuardDuty does not do is protect against denial-of-service or distributed denial-of-service attacks. That's for AWS Shield Standard and Shield Advanced. Also, there's no configuration with GuardDuty. You just turn it on and you're charged according to your usage of Amazon Web Services.

GuardDuty utilizes proprietary machine learning and artificial intelligence (AI) along with strategic partners like CrowdStrike, Splunk, Trend Micro, Trustwave, Rapid7, and Alert Logic. It detects suspicious or unexpected behavior and generates a finding, which is a notification that has the details about an imminent security issue. The finding details include information about what occurred, what AWS resources were involved, for example S3, EC2, or disabling CloudTrail logging in the suspicious activity. It also has a date and timestamp of when the activity took place along with other metadata. The finding type provides a description of the potential security issue. For example, in the reconnaissance finding category after the colon, we see the particular service involved, EC2, and then the finding port probe of an unprotected port. And GuardDuty now uses AI to conduct "pre-crime" identification of bad actors 7-10 days in advance of a malware release or the aforementioned zero day.
