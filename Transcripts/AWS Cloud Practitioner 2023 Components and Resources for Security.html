AWS Cloud Practitioner 2023: Components & Resources for Security

AWS has more than 200 fully featured services for a wide range of technologies, industries, and use cases. Fortunately, the cloud practitioner does not have to know about all of them. There are however quite a few security services to be aware of in this domain. In this course, you will explore AWS security capabilities, including AWS Certificate Manager (ACM), AWS CloudHSM, Amazon Cognito, Amazon Detective, just to name a few. Then you will discover where to access AWS security information, for example, AWS Knowledge Center or AWS Security Blog. Next, you will learn how to manage network access control lists and examine the web application firewall (WAF) and AWS Shield. Finally, you will compare AWS Firewall Manager to AWS Network Firewall and AWS Trusted Advisor to AWS Health Dashboard. This course is one of a collection that prepares learners for the AWS Certified Cloud Practitioner (CLF-C02) exam.

In this first lesson, we're going to conduct a pretty lengthy survey of a number of security services you need to know about for the Cloud Practitioner exam. Now by no means is this a comprehensive list of all the different services at AWS, but fortunately for the Cloud Practitioner, you don't have to be aware of every service at AWS. In fact, some of these services we will return to and discuss again later in the training or we've already briefly touched on them earlier in this training.

Here's AWS Artifact. Artifact is a console-based, self-service auditing object retrieval service that gives customers quick and simple access to AWS compliance documentation and agreements. On the exam remember, there are two different categories of artifact information. One are Artifact Agreements, which allow customers to examine, approve, and manage agreements in AWS Organizations. Then we have Artifact reports where you can get compliance reports from third-party auditors who have tested and verified that AWS is compliant with a variety of global, regional, and industry-specific security standards and regulations. AWS Audit Manager allows customers to continuously map their compliance requirements to AWS usage data with prebuilt and custom frameworks and automated evidence collection.

As the organization works to meet audit and regulatory obligations, they can save time by incorporating audit compliance processes into their DevOps model. That's just one use case. Audit Manager creates an HTTPS or TLS API endpoint to accomplish its solutions. Next, we have the AWS Certificate Manager. The ACM is one of our vital security services. AWS customers can leverage the ACM to provision, manage, and deploy public and private TLS certificates for use with AWS services and even internal connected resources in a hybrid cloud environment. ACM removes the time-consuming, manual process of acquiring, loading, and renewing TLS certificates. ACM supports domain-validated public certificates, private certificates from the AWS Private CA, certificate authority and imported public or private certificates.

A hardware security module is a specialty hardened device that generates keys, stores keys, rotates keys, and supports overall corporate key management. Cloud HSM is a cloud-based version of that. So for example, the customer can use the AWS console and/or the command line interface to provision and manage their own virtual HSMs in the AWS Cloud or they can integrate AWS Cloud HSM Software Development Kit SDK into their applications and then perform cryptographic operations. AWS Cloud HSM enables the customer to provision and manage their own single-tenant hardware security modules, HSMs, allowing them to generate and use their own encryption keys. Then they can use other services like Amazon CloudWatch to log for monitoring and utilization and health of the HSM cluster in real time, or AWS CloudTrail which allows you to audit logs for provisioning, backup, and restore of HSMs uploaded to the customer account.

Amazon Detective. Amazon Detective streamlines the investigative process and assists security teams in conducting quick and effective eDiscovery initiatives. Customers can quickly analyze and determine the nature and extent of possible security issues with Amazon Detective prebuilt data aggregations, summaries, and context. Detective automatically gathers log data from AWS resources, and that uses machine learning (ML), statistical analysis, and graph theory to generate a linked dataset to drive more efficient security inquiries or investigations. A lot of developers at Amazon Web Services want to develop mobile apps. Often they'll give the prospective customer of their mobile app the ability to sign up or create an account. They can do that with Amazon Cognito by creating a user pool.

[Video description begins] A screenshot of the Amazon Cognito user interface is displayed. It showcases a page titled 'Authentication providers'. Below, some information is provided. A series of tabs appear next: Cognito, Amazon, Apple, Facebook, Google+, Twitter/Digits, OpenID, SAML, and Custom. The Cognito tab is active. It displays 2 fields: User Pool ID and App client ID. A tab labeled Add Another Provider appears below. At the bottom of the page, 2 tabs appear: Cancel and Create Pool. [Video description ends]

With Amazon Cognito, AWS developers can add user sign-up and sign-in features with control access to their web and mobile applications. Cognito offers an identity store that scales to millions of users, but it also supports social and enterprise identity federation or single sign-on. So, if the potential customer wants to instead do single sign-on through their Amazon account or their Apple ID, or Facebook or Gmail or Twitter, Amazon Cognito gives the developer an elegant, user-friendly wizard walkthrough to create this environment. It's also built on open identity standards like OpenID and OAuth or SAML 2.0. It also supports various compliance regulations and integrates with front-end and back-end development resources. It also works with AWS Lambda to provide a wide variety of functions that can be used dynamically. For example, presenting your potential customer with a CAPTCHA or a two-factor authentication without having to run a persistent server.

It just simply triggers a Lambda function when the user signs up. Amazon GuardDuty is a managed threat detection service that continually monitors for malicious activity and unauthorized behavior to protect AWS accounts, workloads, and data stored in Amazon S3, for example. GuardDuty can also monitor for zero-day activities. This is malware that just shows up on its first day. It can produce well-defined "findings". It uses proprietary machine learning, an AI algorithm even now involving pre-crime and analysis and prediction, to determine malicious websites and domains 7 to 10 days in advance of launching an attack. Amazon Inspector might seem like it's the same thing as Amazon Detective, but they are completely different services. Amazon Inspector is an automated vulnerability management service that continually scans AWS workloads for software weaknesses and inadvertent network exposures. Inspector offers continual scanning, a maintained vulnerability database, not unlike the national vulnerability database, and auto-discovery of near real-time results.

Inspector contextualizes the findings to generate a meaningful Inspector score. And it uses the detailed findings to automate activities such as ticketing to the service desk or the help desk and automated remediation. AWS Directory Service for Microsoft Active Directory is also known as AWS Managed Microsoft AD. This basically activates your directory-aware workloads and AWS resources to use Managed Active Directory in the AWS Cloud. So, you create a managed directory by quickly creating an AWS Managed Microsoft AD on AWS with just a few clicks in the management console. Or you could use an API in the command line interface. You then deploy or migrate your apps, deploy new apps or migrate existing apps AD-aware workloads to Amazon Web Services, then activate the AWS ecosystem.

Quickly integrate with directory-aware services like EC2, RDS for SQL Server, Amazon Workspaces, and the AWS IAM Identity Center for federated single sign-on. You can manage access by creating users, groups, and roles to manage your access to your directory-aware workloads and AWS services and connect and use your application and AWS services with the existing or new Active Directory credentials. AWS Key Management Service is a critical security service at Amazon Web Services. It lets customers create, manage, and control cryptographic keys across most applications and services. AWS KMS can encrypt data across AWS workloads, digitally sign data, encrypt within applications using the AWS Encryption SDK, and generate and verify message authentication codes or MACs.

The customer managed keys (CMKs) never leave the AWS FIPS 140-validated HSMs unencrypted. In other words, AWS KMS is supported by hardware security modules in the data centers of Amazon Web Services. And yet customers have complete control over permissions, administration, and usage across AWS services. Amazon Macie is a data security service that uses machine learning and pattern matching and recognition to discover and help protect customer's sensitive data. Macie discovers sensitive data and provides visibility into data security risks, allowing automated protection against those risks. Many customers will get visibility into sensitive data across S3 buckets to enhance automated remediation of data security risks. Another common place to run Macie is behind an API gateway for pattern matching to protect data in transit as it interacts with server based and serverless technologies such as AWS Fargate and AWS Lambda.

AWS Resource Access Manager or AWS RAM. Customers use AWS RAM to manage access to resources by creating a resource share. First, they choose the resource share details like a name and resource type, and then they assign it to the resources. Next, they associate permissions. In other words, the actions that can be performed on each resource type in the resource share. They grant access to principals by designating organizational units OUs, AWS accounts, and IAM role and users. When they're done, they'll review and create as their resources are now accessible according to the specified permissions.
